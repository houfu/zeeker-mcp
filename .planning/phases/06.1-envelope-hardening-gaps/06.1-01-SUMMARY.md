## Accomplishments

- **Finding #1 (D6.1-01) closed.** `MetadataCache.license_for_sync()` cold-cache branch now falls back to `config.LICENSES.get(database, ("", ""))` — same chain the warm-but-empty branch already uses. Previously the `return ("", "")` short-circuit surfaced empty `license` and `license_url` on every response served before `_ensure_fresh()` first ran. `list_databases` rows on a freshly-started server now ship the operator-default `CC-BY-4.0` license metadata immediately, not after the first metadata-cache hit.
- **Finding #2 (D6.1-02) closed.** When `query_table` / `fetch` callers narrow `columns=` past a CITATION_TEMPLATES placeholder column, the server now silently appends those placeholder columns to the upstream `_col=` SELECT and strips them from each row dict AFTER `synthesize_citation` reads them. `_citation` is no longer rendered as the empty-placeholder form `"  (, ) — "` for narrow-columns queries. The agent never sees augmented columns at the row top level — `set(row.keys()) == {"_citation", "retrieved_content"}` for `query_table(zeeker-judgements, judgments, columns=["content_text"], limit=1)`. The new `placeholder_columns(template)` helper in `core/citation.py` is the canonical extractor.
- **Finding #4 (D6.1-03) closed.** `query_table(sg-gov-newsrooms, mlaw_news, columns=["content_text"], limit=1)` no longer returns `upstream_unavailable`. Investigation revealed the upstream Datasette deployment has every `sg-gov-newsrooms.*_news` table configured with a default sort on `published_date`; Datasette's auto-generated SQL becomes invalid (`HTTP 400 "Invalid SQL: incomplete input"`) when `_col=` excludes the implicit sort column. The minimal fix injects `_sort=rowid` whenever the caller does not specify a sort — uniform across every table, no behavioral change on tables without configured default sort (e.g. zeeker-judgements.judgments still uses `ORDER BY rowid`). A new ZEEKER_LIVE=1-gated regression test (`tests/test_heavy_column_upstream.py`) catches future regressions.
- **Finding #3 (D6.1-04) closed.** `tests/test_metadata_cache.py::test_live_metadata_parseable` is rewritten as `@pytest.mark.parametrize`-ized across all 4 `ALLOWED_DATABASES`, asserting only that `get_database_license(db)` returns a string OR None — never the empty string (which would mask D6.1-01). A separate non-live `test_policy_license_unaffected_by_upstream_drift` test parametrized over 4 drift cases (literal upstream values, None, an arbitrary string) stubs `/-/metadata.json` with drifted values, warms the cache to verify the warm-cache layer observes the drift, then dispatches `query_table` and asserts `retrieved_content._policy.license` equals the operator-locked `config.CONTENT_POLICIES[(db, table)]["license"]` byte-identical — the dual-layer contract Phase 6 set up.

## Decisions Made

- **D6.1-03 root cause and minimal fix location.** The bug surfaced as `upstream_unavailable` on heavy-column queries against `sg-gov-newsrooms.*_news`. Direct curl probing revealed `_col=` without the table's implicit sort column produces `HTTP 400 "Invalid SQL: incomplete input"` from upstream Datasette. Verified via `GET .../mlaw_news.json?_shape=objects&_col=content_text&_size=1 → HTTP 400`; same URL with `&_sort=rowid` appended → HTTP 200, SQL `select rowid, content_text from mlaw_news order by rowid limit 2`. Every `sg-gov-newsrooms.*_news` table reproduces; `zeeker-judgements.judgments` does NOT (no upstream default-sort). The minimal fix is `_sort=rowid` injection in Step 8 — uniform per-table override, no `HEAVY_COLUMNS` or `LIGHT_COLUMNS` config touch needed (which would have been a scope-boundary expansion).
- **Per-row `_citation` underscore prefix preserved.** Plan 06-02 SUMMARY Deviations §1 established `_citation` (underscore prefix) as the canonical key — necessary because the `judgments` upstream table has a real column literally named `citation`. Plan 06.1-02's augmentation logic preserves this convention end-to-end; the strip step only removes augmented columns (the row's pre-existing `_citation` key is untouched).
- **`tests/tools/test_query_table.py` assertion update kept in-scope.** The pre-existing `test_columns_allowlist_passed_as_repeated_col_keys` asserted `_col == ["citation", "case_name"]` when caller passed `columns=["citation", "case_name"]` — that snapshot pinned the buggy pre-augmentation behavior. Updated to reflect the new D6.1-02 contract (caller columns first, then sorted augmented placeholders). Treated as Rule 1 (bug fix) under the deviation rules: the test was asserting incorrect post-D6.1-02 behavior. Confirmed via grep that this is the ONLY test that pins exact `_col` ordering on a narrow-columns caller.
- **Live regression test binds caches manually.** The `mcp_client` fixture wraps `mcp` (the FastMCP instance) directly via in-memory transport — it does NOT run the production `app.py` lifespan, so DatasetteClient and MetadataCache are unbound at dispatch time. The new `tests/test_heavy_column_upstream.py::test_mlaw_news_heavy_column_returns_content` binds DatasetteClient + MetadataCache + ParentPKCache + tool_started_at manually pointing at `config.UPSTREAM_URL` and calls `query_table` directly as a Python function. This is the only way to exercise the production code path against live upstream without standing up the full Starlette app via `live_server`.
- **Dual-layer invariant test rejects the tautological shape.** A test that compared `config.CONTENT_POLICIES[key]["license"]` against itself would be tautological — the plan's executor note explicitly demanded the stubbed-dispatch form. The implementation stubs `/-/metadata.json` with a DRIFTED upstream license, force-refreshes the cache to verify the warm-cache layer DOES observe the drift, then dispatches `query_table` and asserts `_policy.license` is unaffected. A future bug that accidentally couples upstream license into `_policy` emission would re-trip the gate at the warm-cache assertion AND at the `_policy.license` assertion.
